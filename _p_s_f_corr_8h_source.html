70  struct HSMParams {
71  /*
72  * @brief Parameters that determine how the moments/shape estimation routines make
73  * speed/accuracy tradeoff decisions.
74  *
75  * @param nsig_rg A parameter used to optimize convolutions by cutting off the galaxy
76  * profile. In the first step of the re-Gaussianization method of PSF
77  * correction, a Gaussian approximation to the pre-seeing galaxy is
78  * calculated. If re-Gaussianization is called with the flag 0x4 (as
79  * is the default), then this approximation is cut off at nsig_rg
80  * sigma to save computation time in convolutions.
81  * @param nsig_rg2 A parameter used to optimize convolutions by cutting off the PSF
82  * residual profile. In the re-Gaussianization method of PSF
83  * correction, a "PSF residual" (the difference between the true PSF
84  * and its best-fit Gaussian approximation) is constructed. If
85  * re-Gaussianization is called with the flag 0x8 (as is the default),
86  * then this PSF residual is cut off at nsig_rg2 sigma to save
87  * computation time in convolutions.
88  * @param max_moment_nsig2 A parameter for optimizing calculations of adaptive moments by
89  * cutting off profiles. This parameter is used to decide how many
90  * sigma^2 into the Gaussian adaptive moment to extend the moment
91  * calculation, with the weight being defined as 0 beyond this point.
92  * i.e., if max_moment_nsig2 is set to 25, then the Gaussian is
93  * extended to (r^2/sigma^2)=25, with proper accounting for elliptical
94  * geometry. If this parameter is set to some very large number, then
95  * the weight is never set to zero and the exponential function is
96  * always called. Note: GalSim script devel/modules/test_mom_timing.py
97  * was used to choose a value of 25 as being optimal, in that for the
98  * cases that were tested, the speedups were typically factors of
99  * several, but the results of moments and shear estimation were
100  * changed by <10^-5. Not all possible cases were checked, and so for
101  * use of this code for unusual cases, we recommend that users check
102  * that this value does not affect accuracy, and/or set it to some
103  * large value to completely disable this optimization.
104  * @param regauss_too_small A parameter for how strictly the re-Gaussianization code treats
105  * small galaxies. If this parameter is 1, then the re-Gaussianization
106  * code does not impose a cut on the apparent resolution before trying
107  * to measure the PSF-corrected shape of the galaxy; if 0, then it is
108  * stricter. Using the default value of 1 prevents the
109  * re-Gaussianization PSF correction from completely failing at the
110  * beginning, before trying to do PSF correction, due to the crudest
111  * possible PSF correction (Gaussian approximation) suggesting that
112  * the galaxy is very small. This could happen for some usable
113  * galaxies particularly when they have very non-Gaussian surface
114  * brightness profiles -- for example, if there's a prominent bulge
115  * that the adaptive moments attempt to fit, ignoring a more
116  * extended disk. Setting a value of 1 is useful for keeping galaxies
117  * that would have failed for that reason. If they later turn out to
118  * be too small to really use, this will be reflected in the final
119  * estimate of the resolution factor, and they can be rejected after
120  * the fact.
121  * @param adapt_order The order to which circular adaptive moments should be calculated
122  * for KSB method. This parameter only affects calculations using the
123  * KSB method of PSF correction. Warning: deviating from default
124  * value of 2 results in code running more slowly, and results have
125  * not been significantly tested.
126  * @param max_mom2_iter Maximum number of iterations to use when calculating adaptive
127  * moments. This should be sufficient in nearly all situations, with
128  * the possible exception being very flattened profiles.
129  * @param num_iter_default Number of iterations to report in the output ShapeData structure
130  * when code fails to converge within max_mom2_iter iterations.
131  * @param bound_correct_wt Maximum shift in centroids and sigma between iterations for
132  * adaptive moments.
133  * @param max_amoment Maximum value for adaptive second moments before throwing
134  * exception. Very large objects might require this value to be
135  * increased.
136  * @param max_ashift Maximum allowed x / y centroid shift (units: pixels) between
137  * successive iterations for adaptive moments before throwing
138  * exception.
139  * @param ksb_moments_max Use moments up to ksb_moments_max order for KSB method of PSF
140  * correction.
141  * @param failed_moments Value to report for ellipticities and resolution factor if shape
142  * measurement fails.
143  */
